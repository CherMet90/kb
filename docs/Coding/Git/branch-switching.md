## Переключаемся на ветку с переносом данных на неё
```
git checkout main           # переключаемся на желаемую ветку
git pull origin main        # синхронизируем данные с удаленным репо
git merge release/2.1.0     # переносим файлы проекта из ветки (release -> main, в данном случае)
git tag v2.1.0              # опционально - тегируем релиз (тег начнет, например, отображаться в разделе релизы на гитхабе)
git push origin main --tags # пушим в удаленный репо
```
